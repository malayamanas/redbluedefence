# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Overview

This is a **pre-built static site** for [RedBlueDefence](https://www.redbluedefence.com), a cybersecurity company. The repository contains only the compiled HTML/CSS/JS output — there are no Hugo source files, templates, or build configuration here. The site was generated by Hugo 0.150.0.

Deployment happens via **GitHub Pages** (`git@github.com:malayamanas/redbluedefence.git`). Pushing to `main` deploys the site. The `CNAME` file routes `www.redbluedefence.com` to GitHub Pages.

## Site Structure

The URL structure mirrors the directory structure exactly:

| Directory | Purpose |
|-----------|---------|
| `index.html` | Homepage |
| `insight/` | Blog-style articles (AI threat detection, employee training) |
| `case-studies/` | Client case studies (data breach prevention, quantum cryptography) |
| `careers/` | Job listings (cloud security architect, penetration tester) |
| `services/` | Services page |
| `company/` | About/company page |
| `contact/` | Contact page |
| `categories/` | Tag/category index pages (cybersecurity, technology, tech-verse) |
| `faqs/`, `reviews/`, `privacy-policy/`, `terms-and-conditions/` | Supporting pages |

Each section has:
- `index.html` — rendered page
- `index.xml` — RSS feed (content sections only)

## Assets

- `css/style.min.[hash].css` — main stylesheet (loaded immediately)
- `css/style-lazy.min.[hash].css` — non-critical styles (lazy-loaded via print media trick)
- `js/script.min.[hash].js` — main JavaScript
- `js/script-lazy.min.[hash].js` — deferred JavaScript
- `images/` — all site images including `logo.svg`, `favicon_*.png`, `og-image.png`
- `sitemap.xml` — SEO sitemap

CSS and JS files use content-hash filenames. When assets change, **all HTML files referencing them must be updated** to point to the new hash.

## Editing Workflow

Since this is committed build output with no build system:

1. **Edit HTML directly** — changes to page content, links, images, and meta tags are made in the relevant `index.html` files.
2. **Commit and push to `main`** to deploy changes live.

When fixing URLs or links, they typically appear in `href`, `src`, and `<link>` attributes throughout the HTML. Use Grep to find all occurrences before editing.

## Key External Dependencies (CDN)

- **AOS (Animate On Scroll):** `https://unpkg.com/aos@2.3.1/dist/aos.css`
- **Google Fonts:** Manrope and Rethink Sans families

## Content Conventions

- Primary brand color: `#7c43ff` (purple gradient)
- Font families: Manrope (body), Rethink Sans (headings)
- CSS utility classes follow a Tailwind-like convention (`flex`, `items-center`, `text-white`, etc.)
- AOS animations use `data-aos="fade-down"` attributes on sections
- Navigation uses a CSS checkbox toggle (`#nav-toggle`) for mobile menu — no JS required for open/close
